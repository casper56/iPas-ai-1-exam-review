### 1. 頁面版面與 UI 設計 (Layout & UI)
- **整體風格**：採用現代簡潔、低飽和度的深色或淺色調（適合長時間閱讀），整體排版需乾淨易讀。
- **基本資訊**：頁面頂部需包含「試卷標題」與「考試日期」等基本資訊欄位。
- **題目排版 (Flexbox/Grid)**：
  - 每一個題目區塊包含：旗標按鈕、題號、題目內容、答案顯示欄。這些元素必須採「橫向並排」排列。
  - 題目下方的單選/多選選項，必須以「兩欄式 (2-column layout)」格線呈現。
  - 有一sidemenu, 紀錄每個題目答題狀況,正確:淺綠, 錯誤:玫瑰紅, 更正:橘色

### 2. 答案隱藏與互動機制 (Answer Masking)
- **隱藏邏輯**：正確答案的文字顏色必須與背景顏色完全相同（例如白底白字），在視覺上達到完全隱藏的效果。
- **互動顯示**：
  - 當使用者將滑鼠指針移入（Hover）該題目區塊時，答案文字需自動變色（或顯示背景色）顯現出來。
  - 允許使用者透過滑鼠「反白選取（Selection）」答案欄位的文字來查看答案。

### 3. 旗標與篩選功能 (Flagging & Filtering)
- **容易出錯標記**：每題左側的旗幟按鈕，點擊後可將該題切換為「容易出錯」狀態。
- **視覺提示**：被標記的題目，其外框需變成醒目的顏色（如淡紅色或橘色邊框），並加上「易錯題」的微型標籤。
- **狀態持久化**：使用瀏覽器的 `localStorage` 來儲存每題的標記狀態。頁面重新整理或關閉後再度開啟時，標記狀態必須完整保留。
- **上方控制面板**：
  - 提供「全部題目」與「已標記」兩個 Tab 或 Button 進行切換。點擊「已標記」時，頁面只顯示被標記為易錯的題目。
  - 即時統計並顯示「目前已標記的總題數」。

### 4. 程式碼實作要求
- 請將所有的 HTML 結構、CSS 樣式表（建議使用 Flexbox/Grid 佈局，或引入 CDN Tailwind CSS）、以及處理儲存與篩選邏輯的原生 JavaScript 程式碼，全部整合在同一個 `.html` 檔案中。
- JavaScript 邏輯需在 DOM 加載完成後，讀取 `localStorage` 並動態為被標記的題目加上對應的 CSS 類別（Class）。
